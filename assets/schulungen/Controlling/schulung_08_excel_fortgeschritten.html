01

Power Query — Daten automatisch aufbereiten

Power Query ist das wichtigste Werkzeug für alle, die regelmäßig Daten aus SAP, ERP-Systemen, CSV-Exporten oder mehreren Excel-Dateien zusammenführen müssen. Statt monatlich dieselben Schritte manuell zu wiederholen, werden sie einmalig definiert und dann per Klick wiederholt.

Warum Power Query statt manueller Aufbereitung?

  • Reproduzierbarkeit: Jede Transformation wird aufgezeichnet und ist jederzeit nachvollziehbar.
  • Geschwindigkeit: Monatlicher Report-Refresh in Sekunden statt Stunden.
  • Fehlerreduktion: Kein Copy-Paste, keine vergessenen Filter, keine Formatfehler.
  • Skalierbarkeit: Funktioniert mit Millionen von Zeilen — Excel-Tabellenblätter haben ein Limit von ~1 Mio. Zeilen.

Der Power Query Editor — Schritt für Schritt

  1. Daten laden: Daten → Daten abrufen → Aus Datei/Datenbank/Web. Power Query öffnet den Editor mit einer Vorschau.

  2. Spalten bereinigen: Datentypen zuweisen, Leerstellen entfernen, Duplikate löschen, Spalten umbenennen. Jeder Schritt erscheint im Panel "Angewendete Schritte" rechts.

  3. Transformationen anwenden: Spalten zusammenführen, aufteilen, berechnen. Datumsspalten in Jahr/Monat zerlegen. Texte standardisieren.

  4. Tabellen zusammenführen (Merge/Append): Mehrere Quelltabellen nach Schlüsselspalten verbinden — wie ein SQL-JOIN, ohne SQL zu kennen.

  5. Laden & Aktualisieren: Schließen & Laden → Die aufbereiteten Daten landen in Excel. Bei neuen Quelldaten: Rechtsklick → Aktualisieren. Fertig.

Häufige Power Query Transformationen im Controlling

M-Sprache: Im Hintergrund schreibt Power Query automatisch Code in der Sprache "M". Du musst kein M kennen — aber wenn du den generierten Code siehst (Start → Erweiterter Editor), beginnt das Verständnis und du kannst Schritte gezielt anpassen.

Beispiel: Einfache M-Formel zur Monatsnummer

// Automatisch generierter M-Code für eine Datumsextraktion: let Quelle = Excel.CurrentWorkbook(){[Name="tbl_Buchungen"]}[Content], Typ = Table.TransformColumnTypes(Quelle,{{"Datum", type date}}), MonatHinzu = Table.AddColumn(Typ, "Monat", each Date.Month([Datum]), Int64.Type), JahrHinzu = Table.AddColumn(MonatHinzu, "Jahr", each Date.Year([Datum]), Int64.Type) in JahrHinzu

Jeder Schritt ist im Panel "Angewendete Schritte" sichtbar und einzeln rückgängig zu machen. Das macht Power Query deutlich transparenter als VBA-Makros.

02

Datenmodelle & Power Pivot

Wenn mehrere Tabellen miteinander in Beziehung stehen — zum Beispiel Buchungen, Kostenstellen und Kontenrahmen — wird ein Datenmodell gebraucht. Power Pivot (in Excel integriert) ermöglicht relationale Auswertungen ohne SVERWEIS-Ketten.

Star-Schema: Das Fundament des Datenmodells

Im Controlling-Kontext orientiert man sich meist am Star-Schema:

  • Faktentabelle (Mitte): Transaktionale Daten mit vielen Zeilen — z.B. alle Buchungen mit Betrag, Datum, Kostenstellen-ID, Konto-ID.
  • Dimensionstabellen (Äste): Stammdaten mit wenigen Zeilen — Kostenstellen, Konten, Perioden, Mitarbeiter.
  • Beziehungen: Verbindungen zwischen Fakten- und Dimensionstabellen über Schlüsselspalten (1:n).
Vorteil gegenüber SVERWEIS: Im Datenmodell werden Tabellen einmal verknüpft — danach können alle Pivot-Tabellen auf jede Kombination aus Fakten und Dimensionen zugreifen, ohne dass Spalten kopiert oder SVERWEIS-Formeln gepflegt werden müssen.

DAX — Berechnungen im Datenmodell

DAX (Data Analysis Expressions) ist die Formelsprache für Power Pivot und Power BI. Die Syntax ähnelt Excel, aber DAX arbeitet mit ganzen Tabellenspalten statt einzelnen Zellen.

// Einfache DAX-Measure: Umsatz gesamt Umsatz = SUM(tbl_Buchungen[Betrag]) // Plan-Ist-Abweichung in % Abweichung % = DIVIDE([Ist_Umsatz] - [Plan_Umsatz], [Plan_Umsatz], 0) // Vorjahreszeitraum-Vergleich (Time Intelligence) Umsatz Vorjahr = CALCULATE([Umsatz], SAMEPERIODLASTYEAR(dim_Datum[Datum])) // Laufende Jahressumme (YTD) Umsatz YTD = TOTALYTD([Umsatz], dim_Datum[Datum])

Wann Datenmodell statt normaler Pivot?

SituationEmpfehlung
Eine Tabelle, < 100.000 ZeilenNormale Pivot-Tabelle reicht völlig aus
Mehrere Tabellen mit Beziehungen→ Datenmodell / Power Pivot
Vorjahresvergleich, YTD-Werte nötig→ DAX Time Intelligence im Datenmodell
> 1 Mio. Zeilen oder Verknüpfung mit Datenbank→ Power BI (auf Power Pivot aufbauend)
03

Dynamische Diagramme

Statische Diagramme zeigen immer denselben Ausschnitt. Dynamische Diagramme passen sich an Filterauswahl, Zeitraum oder Dateneingabe automatisch an — ohne dass der Nutzer in den Chart eingreifen muss.

Methode 1: Pivot-Diagramm mit Slicer

Der einfachste Weg zu einem interaktiven Diagramm: Pivot-Tabelle erstellen → PivotChart einfügen → Slicer hinzufügen. Der Slicer filtert gleichzeitig Tabelle und Chart.

  1. Pivot-Tabelle auf Basis der Datentabelle oder des Datenmodells erstellen.

  2. Pivot-Tabelle markieren → PivotTable-Analyse → PivotChart → Diagrammtyp wählen.

  3. PivotTable-Analyse → Slicer einfügen → Felder auswählen (z.B. Jahr, Kostenstelle, Produkt).

  4. Slicer formatieren: Rechtsklick → Größe und Eigenschaften → Spaltenanzahl anpassen.

Methode 2: BEREICH.VERSCHIEBEN für dynamische Datenquellen

Mit der Funktion BEREICH.VERSCHIEBEN (OFFSET) lässt sich der Datenbereich eines Diagramms dynamisch an eine Dropdown-Auswahl koppeln.

// Dynamischer Named Range für Diagramm-Datenbasis // Name: "DynUmsatz" — bezieht sich auf ausgewählten Monat =BEREICH.VERSCHIEBEN(Daten!$B$2; 0; VERGLEICH($E$1; Daten!$B$1:$M$1; 0)-1; ANZAHL2(Daten!$A:$A)-1; 1)

Methode 3: Dynamische Arrays + Diagramm (Excel 365)

In Excel 365 können dynamische Arrayfunktionen die Diagrammbasis automatisch anpassen:

// FILTER: Nur Zeilen einer bestimmten Kostenstelle anzeigen =FILTER(A2:D200; C2:C200=$G$1; "Keine Daten") // SORTIEREN: Ergebnisse für Diagramm absteigend sortieren =SORTIEREN(FILTER(A2:D200; C2:C200=$G$1); 4; -1) // EINDEUTIG: Alle Kostenstellen für Dropdown-Liste =EINDEUTIG(C2:C200)
TIPP
Diagramm-Titel dynamisch
Klick auf Diagrammtitel → in Bearbeitungsleiste =E1 eingeben (E1 = Zelle mit aktuellem Filter). Titel ändert sich automatisch mit dem Slicer.
TIPP
Sparklines für Mini-Charts
Einfügen → Sparklines → Linien/Balken/Gewinn-Verlust. Ideal für Trenddarstellung direkt in der Zeile einer KPI-Tabelle. Braucht kaum Platz.
TIPP
Sekundärachse für Plan-Ist
Ist-Werte als Balken, Plan-Werte als Linie mit Sekundärachse. Rechtsklick auf Datenreihe → Datenreihen-Diagrammtyp ändern → Linie → Sekundärachse.
TIPP
Wasserfall-Diagramm
Excel 2016+ enthält den Diagrammtyp "Wasserfall" (Waterfall). Ideal für Abweichungsbrücken: Plan → Einzeltreiber → Ist.
04

LAMBDA-Funktionen — eigene Formeln schreiben

LAMBDA (verfügbar in Excel 365 ab 2021) ist ein Meilenstein: Erstmals können Controller eigene, wiederverwendbare Funktionen ohne VBA schreiben. Eine LAMBDA-Funktion nimmt Parameter entgegen und führt eine Berechnung durch — wie eine Mini-Programmiersprache innerhalb von Excel.

Syntax und Grundprinzip

// Allgemeine Syntax: =LAMBDA(Parameter1; Parameter2; ...; Formel) // Beispiel 1: Einfache LAMBDA direkt aufrufen (zwei Parameter) =LAMBDA(preis; menge; preis * menge)(100; 50) // → 5000 // Beispiel 2: Als benannte Funktion im Namens-Manager speichern // Name: "DECKUNGSBEITRAG" // Bezieht sich auf: =LAMBDA(preis; var_kosten; menge; (preis - var_kosten) * menge) // Dann in jeder Zelle verwendbar: =DECKUNGSBEITRAG(150; 90; 200) // → 12.000

Praxisbeispiele für Controller

// PLANIST: Relative Plan-Ist-Abweichung =LAMBDA(ist; plan; WENNFEHLER((ist-plan)/plan; 0)) // AMPEL: Gibt "🟢", "🟡" oder "🔴" zurück =LAMBDA(abw_pct; WENN(ABS(abw_pct) < 0,05; "🟢"; WENN(ABS(abw_pct) < 0,10; "🟡"; "🔴"))) // JAHRESKUM: Kumuliert alle Werte bis zum aktuellen Monat =LAMBDA(bereich; monat; SUMME(INDEX(bereich; 1; 1):INDEX(bereich; 1; monat)))

Interaktive LAMBDA-Demo: Deckungsbeitragsrechner

🧪 LAMBDA Simulation — =DECKUNGSBEITRAG(preis; var_kosten; menge)

Ergebnis: 12.000 €

=DECKUNGSBEITRAG(150; 90; 200) → (150−90)×200 = 12.000 €

LAMBDA mit LET kombinieren

// LET ermöglicht benannte Zwischenvariablen innerhalb von LAMBDA =LAMBDA(umsatz; var_kosten; fix_kosten; LET( db; umsatz - var_kosten; ebit; db - fix_kosten; marge; WENNFEHLER(ebit/umsatz; 0); marge // Rückgabewert ) )
Voraussetzung: LAMBDA und dynamische Arrayfunktionen (FILTER, SORTIEREN, EINDEUTIG) sind nur in Excel 365 / Excel 2021 verfügbar. In älteren Versionen funktionieren diese Formeln nicht. Vor der Einführung in Firmen-Templates Versionsstand der Nutzer prüfen.
05

Automatisierte Reports

Das Ziel fortgeschrittener Excel-Nutzung ist nicht mehr Formeln beherrschen — es ist den Prozess zu eliminieren. Ein vollautomatisierter Report aktualisiert sich mit einem Klick auf "Alle aktualisieren".

Architektur eines automatisierten Excel-Reports

SchichtInhaltWerkzeug
1. DatenquelleSAP-Export, CSV, Datenbank, SharePointPower Query Verbindung
2. AufbereitungBereinigung, Transformation, MergePower Query
3. BerechnungKPIs, Abweichungen, ZeitreihenDatenmodell / DAX oder Formeln
4. DarstellungCharts, KPI-Widgets, TabellenPivot + Slicer + bedingte Formatierung
5. RefreshAlle Schritte neu berechnenDaten → Alle aktualisieren (1 Klick)

Checkliste: Report-Automatisierung

  • ✅ Datenquelle per Power Query eingebunden (kein Copy-Paste)
  • ✅ Alle Transformationen in Power Query dokumentiert (keine manuellen Eingriffe)
  • ✅ Formeln mit WENNFEHLER abgesichert (kein #NV in Reports)
  • ✅ Pivot-Tabellen auf intelligente Tabellen (Strg+T) aufgebaut
  • ✅ Dashboard-Blatt: Gitterlinien aus, Zeilenköpfe aus, Zoom auf 100%
  • ✅ Druckbereich definiert für PDF-Export (Seitenansicht → Druckbereich festlegen)
  • ✅ Datei als .xlsx gespeichert, nicht .xls (moderne Funktionen, bessere Performance)
  • ✅ Schutz der Formeln: Review → Blatt schützen (Eingabezellen freigeben)
Nächster Schritt nach Excel: Wenn ein Report wöchentlich gebraucht wird, komplexe Beziehungen hat und von mehreren Personen gleichzeitig genutzt wird, ist Power BI der natürliche nächste Schritt. Die Konzepte (Power Query, DAX, Star-Schema) sind identisch — nur die Oberfläche wechselt.
