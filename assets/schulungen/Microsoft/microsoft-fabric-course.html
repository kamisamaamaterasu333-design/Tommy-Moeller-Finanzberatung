What is

Simple Analogy

Imagine your office uses 10 different tools: Excel for analysis, a database for storing data, a reporting tool for dashboards, a pipeline tool for moving data. Microsoft Fabric replaces all of them with one platform — everything shares the same storage, the same security, and the same interface.

The Core Idea

Microsoft Fabric is an all-in-one analytics platform built on Microsoft Azure. It was launched in 2023 and combines tools for data engineering, data science, real-time analytics, data warehousing, and business intelligence — bundled into a single product.

Before Fabric, you needed separate products: Azure Synapse, Azure Data Factory, Power BI, Azure Data Lake. Fabric bundles all of these into one — with one price, one interface, one security model.

Module 02 · Storage

OneLake —
One Storage for All

The shared data foundation underneath every Fabric workload. Think of it as a company-wide Google Drive — but for data.

Simple Analogy

Think of OneLake like a shared Google Drive for your whole company's data. Instead of every department having their own storage, everyone saves to the same place. A data engineer puts raw data there, a Power BI report reads from that same file — no copying, no delays, no confusion about which version is correct.

What is OneLake?

OneLake is the single storage layer built into every Fabric workspace. It uses Delta Parquet — a powerful, efficient file format (like a super-charged CSV that supports versioning).

Every piece of data in Fabric — tables, files, reports — lives in OneLake.

Key Benefits
  • ✓No data duplication — one copy, many tools
  • ✓Automatically backed up and versioned
  • ✓Works like a folder structure you know
  • ✓Connects to external sources too
OneLake Structure
# How OneLake is organized Tenant (your company) └── Workspace A # e.g. "Sales Team" ├── Lakehouse # raw + cleaned files ├── Warehouse # structured SQL tables └── Reports # Power BI dashboards └── Workspace B # e.g. "Finance Team" ├── Lakehouse └── Warehouse
Lakehouse

Flexible — stores raw files, CSVs, images, JSON. Great for data engineers. Like a well-organized filing cabinet where you can store anything.

Warehouse

Structured — organized tables with strict schemas, queryable with SQL. Great for analysts. Like a well-structured Excel workbook with rigid column types.

Module 03 · Data Engineering

Pipelines &
ETL Automation

Move and transform data automatically — from any source to your Lakehouse or Warehouse, on a schedule, without manual work.

Simple Analogy

A data pipeline is like a factory conveyor belt. Raw materials (your data) come in on one end, get cleaned and shaped along the way, and a finished product (clean, usable data) comes out the other end — automatically, every time, without anyone manually doing it.

What is ETL?

ETL stands for Extract, Transform, Load — the three steps of every data pipeline:

  • 📤
    Extract — pull data from a source (Excel file, database, website, SharePoint, API)
  • 🔧
    Transform — clean it: remove duplicates, fix formats, calculate new columns, merge tables
  • 📥
    Load — put the clean data into your destination (Lakehouse or Warehouse)
Three Tools for Pipelines
🔗
Data Pipeline NO-CODE
Drag-and-drop interface. Copy data from A to B, schedule it to run daily. No coding needed — perfect starting point for your level.
⚙️
Dataflow Gen2 LOW-CODE
Like Power Query in Excel — a familiar interface to clean and transform data step by step. If you've used Power Query, you already know this.
💻
Spark Notebooks CODE
Python or SQL notebooks for complex transformations. Advanced — come back to this after you're comfortable with the basics.
Your First Pipeline — Step by Step
  1. 1
    Open your Fabric workspace → click New → Data Pipeline
  2. 2
    Add a Copy Data activity — choose your source (e.g. an Excel file on SharePoint)
  3. 3
    Choose your destination — your Lakehouse table in OneLake
  4. 4
    Click Validate, then Run — data flows automatically
  5. 5
    Set a schedule — e.g. run every morning at 6:00 AM

Module 04 · Data Warehouse

Data Warehouse
& SQL Queries

The organized, structured heart of your analytics. If you know basic SQL, you can query your Fabric Warehouse right away.

Simple Analogy

A Data Warehouse is like a very organized library. Every book (table) has a clear title, a fixed structure, and sits in the right section. You can find any information quickly using a search system — SQL. Unlike the Lakehouse (a flexible storage room), the Warehouse is always tidy and strictly organized.

When to Use a Warehouse
  • ✓Your data is structured (rows & columns)
  • ✓Business users need to run SQL queries
  • ✓You need joins between multiple tables
  • ✓You need strict governance and permissions
Key Concepts
  • 📋
    Fact table — your transactions (sales, orders, events)
  • 📖
    Dimension table — context (customers, products, dates)
  • ⭐
    Star schema — fact in center, dimensions around it
SQL in Fabric — Ready to Use
-- Get total sales per customer (familiar SQL) SELECT c.customer_name, SUM(s.amount) AS total_sales, COUNT(*) AS order_count FROM fact_sales s JOIN dim_customers c ON s.customer_id = c.id WHERE s.sale_date >= '2024-01-01' GROUP BY c.customer_name ORDER BY total_sales DESC;

Standard T-SQL — if you know basic SQL already, you can write queries like this in the Fabric Warehouse editor immediately, with no setup required.

Creating Your First Table
-- Create a simple sales table CREATE TABLE fact_sales ( sale_id INT NOT NULL, customer_id INT NOT NULL, product VARCHAR(100) NOT NULL, amount DECIMAL(10,2) NOT NULL, sale_date DATE NOT NULL ); -- Insert some test data INSERT INTO fact_sales VALUES (1, 101, 'Laptop', 999.99, '2024-03-15'), (2, 102, 'Monitor', 349.50, '2024-03-16');

Module 05 · Reporting

Power BI
Inside Fabric

Turn your warehouse data into beautiful, interactive reports — directly in the browser, always showing the latest data.

Simple Analogy

If your Warehouse is the filing cabinet full of data, Power BI is the presentation layer — the beautiful, live slideshow you show your manager. Click on a bar chart and everything else on the page filters instantly. It connects live to your Fabric data, so it's always up to date.

Power BI is Natively Built Into Fabric

In Fabric, you don't need to install Power BI Desktop separately. Create reports directly in the browser, connected live to your Warehouse or Lakehouse. Any change to the underlying data is automatically reflected in your report — no manual refreshes.

Four Key Power BI Concepts
  • 📊
    Semantic Model — defines tables, relationships, and calculated measures. The "brain" behind your reports.
  • 📄
    Report — the visual layer: bar charts, line charts, KPI cards, tables, maps built on the semantic model.
  • 📋
    Dashboard — a pinboard of visuals from multiple reports for a quick executive overview.
  • 🔢
    DAX measure — a formula to calculate values dynamically. Like Excel's SUMIF, but far more powerful.
Your First DAX Measures
-- Total Sales (simple SUM) Total Sales = SUM(fact_sales[amount]) -- Sales Year-to-Date Sales YTD = TOTALYTD( SUM(fact_sales[amount]), dim_date[date] ) -- % Growth vs Last Year Growth % = DIVIDE( [Sales YTD] - [Sales Last Year], [Sales Last Year], 0 -- return 0 if division by zero )

DAX reads like Excel formulas. If you know SUMIF and VLOOKUP, you understand the concept — DAX is just more powerful and context-aware.

DirectLake Mode — Fabric's Superpower

Normally Power BI imports a copy of your data and refreshes it on a schedule. In Fabric, DirectLake mode reads directly from OneLake — no copy, no refresh delay. Your reports always show the latest data, even for billion-row datasets.

Module 06 · Advanced Features

Real-Time Analytics
& Data Science

Process live data streams and build machine learning models — all within the same Fabric platform.

Real-Time Analytics

Fabric can process data as it arrives — every second. Use cases:

  • 📡Live sensor readings from machines
  • 🛒E-commerce orders as they happen
  • 📈Live stock price feeds
  • 🚗Fleet tracking & GPS data

Tool: Eventhouse. Query language: KQL (similar to SQL but for time-series data).

Data Science

Fabric includes Python notebooks to:

  • 🤖Train machine learning models
  • 📊Run statistical analysis
  • 🔮Make predictions (sales forecast)
  • 🧪Experiment & track model results

Uses Apache Spark — scales to massive datasets automatically.

Copilot in Fabric — AI Built In

Microsoft has integrated AI (Copilot) throughout Fabric. You can type in plain language:

You type:
"Create a report showing monthly sales by region for the last 12 months"

Copilot builds the report. You can also ask it to write SQL queries, explain what a pipeline does, or suggest transformations — very useful while you're still learning.

Your 3-Month Study Plan
Month 1 — Foundations
Complete this course in full. Set up your free Fabric 60-day trial. Build your first end-to-end project with a sample dataset. Start the Microsoft Learn "Fabric Analytics Engineer" learning path (free, ~15 hours).
