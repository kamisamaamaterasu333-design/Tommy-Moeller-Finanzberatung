Power Fx – die Formelsprache
Excel-ähnliche Formeln für App-Logik
MittelPower Fx
▼

Power Fx ist die Open-Source-Formelsprache von Power Apps – stark an Excel-Formeln angelehnt. Formeln steuern das Verhalten von Controls, navigieren zwischen Screens und schreiben Daten.

Häufige Formeln
// Datensätze filtern (z.B. nur aktive Kunden) Filter(Kunden, Status = "Aktiv") // Datensatz speichern Patch(Auftraege, Defaults(Auftraege), { Titel: txtTitel.Text, Status: "Neu", Erstellt: Now() }) // Zwischen Screens navigieren Navigate(Screen_Detail, ScreenTransition.Fade) // Bedingte Sichtbarkeit eines Controls If(Rolle.Selected.Value = "Admin", true, false) // Datensatz löschen Remove(Auftraege, Gallery1.Selected)
Szenario
Mitarbeiter füllt Formular aus → App validiert → speichert in SharePoint Liste

Patch() + Notify()
Schreiben + Erfolgsmeldung

Aufbau eines Cloud Flows
Trigger → Aktionen → Bedingungen → Schleifen
Mittel
▼

Jeder Cloud Flow besteht aus genau einem Trigger (was startet den Flow?) und beliebig vielen Aktionen. Steuerelement-Aktionen wie Bedingung, Schleife und Scope ermöglichen Logik.

  • 1
    Trigger wählen
    z.B. „Wenn ein Element in SharePoint erstellt wird" oder „Täglich um 08:00 Uhr"
  • 2
    Dynamische Inhalte nutzen
    Ausgaben des Triggers (z.B. E-Mail-Betreff, Datei-Name) in späteren Schritten verwenden
  • 3
    Bedingungen & Schleifen
    „Condition" für Ja/Nein-Verzweigungen; „Apply to each" für Listen; „Do until" für Warteschleifen
  • 4
    Aktionen ausführen
    E-Mail senden, Dataverse-Datensatz anlegen, Teams-Nachricht, HTTP-Aufruf, …
  • 5
    Fehlerbehandlung
    Scope + „Configure run after" für Try/Catch-Pattern; Flow-History für Debugging
Beispiel: Genehmigungsworkflow
// Trigger: Neuer Datensatz in Dataverse-Tabelle "Urlaubsanträge" Genehmigung senden: Start and wait for an approval → Typ: "Approve/Reject – First to respond" → An: manager@firma.de → Details: "Urlaubsantrag von " + triggerBody/Mitarbeiter Condition: Outcome is equal to "Approve" Ja → Dataverse: Status auf "Genehmigt" setzen + E-Mail an Mitarbeiter: "Urlaub genehmigt ✅" Nein → Status auf "Abgelehnt" setzen + E-Mail mit Begründung
Desktop Flows & RPA
Robotic Process Automation – Legacy-Systeme automatisieren
FortgeschrittenRPA
▼

Desktop Flows automatisieren Aktionen auf dem Windows-Desktop: Klicks, Tastatureingaben, Fensterinteraktionen – auch in Anwendungen ohne API (SAP GUI, alte Windows-Programme, Webseiten ohne API).

ModusBedeutungWann?
AttendedLäuft auf dem aktiven PC des NutzersWenn Benutzerinteraktion nötig (Login, CAPTCHA)
UnattendedLäuft auf einem dedizierten Bot-PCNächtliche Batch-Jobs, vollständig automatisiert
⚠️
Unattended Desktop Flows erfordern ein Premium-Addon (Power Automate Premium oder Process). Für einfache Cloud-Flows reicht oft das M365-Lizenzrecht.

🌐 Power Pages

Power Pages

Externe Websites und Self-Service-Portale – für Kunden, Partner und Bürger.

Was ist Power Pages?
Low-Code Webportale mit Dataverse-Backend
Einstieg
▼

Power Pages (früher Power Apps Portals) ermöglicht den Aufbau öffentlicher oder authentifizierter Websites, die direkt auf Dataverse-Daten zugreifen. Externe Benutzer können sich registrieren, Formulare ausfüllen und Daten einsehen.

Typische Portale
• Kundenportal (Tickets, Status)
• Lieferantenportal
• Bürgerportal (Behörden)
• Partnerportal
Kernfunktionen
• Authentifizierung (AAD, Social)
• Formulare mit Dataverse
• Listenansichten
• Mehrspr. Unterstützung
⚠️
Power Pages ist das einzige Tool der Power Platform, das ausschließlich Dataverse als Datenquelle nutzt. SharePoint oder SQL sind nicht direkt anbindbar – ggf. über Power Automate als Zwischenschicht.
Seitenerstellung & Design Studio
Der Pages Design Studio & Liquid-Templates
Mittel
▼

Das Power Pages Design Studio bietet einen visuellen Editor für Seiten, Themes und Komponenten. Für komplexere Logik wird Liquid verwendet – eine Template-Sprache für dynamische Inhalte.

{% comment %} Liquid-Beispiel: Aktuellen Benutzer begrüßen {% endcomment %} {% if user %} <h2>Hallo, {{ user.fullname }}!</h2> <p>Deine offenen Anfragen: {{ user.incidents.size }}</p> {% else %} <p>Bitte melde dich an.</p> {% endif %}
💡
Für einfache Portale reicht das Design Studio vollkommen aus. Liquid und Web Templates werden erst bei komplexer Darstellungslogik oder individuellen Layouts benötigt.

🤖 Copilot Studio

Copilot Studio

KI-Chatbots und Copiloten erstellen, anpassen und in bestehende Kanäle integrieren.

Was ist Copilot Studio?
Von Power Virtual Agents zu generativer KI
Einstieg
▼

Copilot Studio (früher Power Virtual Agents) ist ein Low-Code-Tool zum Erstellen von KI-Chatbots und zur Anpassung von Microsoft 365 Copilot. Es kombiniert klassische regelbasierte Konversationsflows mit generativer KI (GPT).

ModusBeschreibung
Klassischer BotThemen-basiert: Trigger-Phrasen → vordefinierte Dialog-Trees → Aktionen
Generativer ModusKI beantwortet Fragen direkt aus Wissensdatenquellen (Websites, Sharepoint, Dateien)
M365 Copilot ErweiterungEigene Plugins / Copilot Actions für den Unternehmens-Copiloten erstellen
Aufbau eines Bots: Themen & Topics
Trigger, Dialog-Nodes und Aktionsaufrufe
Mittel
▼
  • 1
    Trigger Phrases definieren
    Welche Nutzeräußerungen starten dieses Thema? z.B. "Urlaub beantragen", "Wo ist meine Bestellung?"
  • 2
    Dialog-Nodes aufbauen
    Fragen stellen, Variablen befüllen, Bedingungen prüfen, Nachrichten ausgeben
  • 3
    Power Automate-Flow aufrufen
    Für Datenbankabfragen, externe API-Calls oder E-Mails → Flow gibt Ergebnis zurück
  • 4
    Kanal publizieren
    Teams, Website, SharePoint, Dynamics 365, Telegram, Slack uvm.
💡
Mit „Generative Antworten" (Generative Answers) kann der Bot Fragen automatisch aus einer SharePoint-Bibliothek oder Website beantworten – ohne manuelles Anlegen von Themen.

🗄️ Dataverse

Dataverse

Die zentrale Datenbankplattform der Power Platform – sicher, skalierbar, tief integriert.

Was ist Dataverse?
Mehr als eine Datenbank – eine Geschäftsdatenplattform
Einstieg
▼

Dataverse ist ein SaaS-Datenbankdienst auf Basis von Azure SQL + Azure Storage. Im Gegensatz zu einer rohen Datenbank bietet Dataverse eingebaute Geschäftslogik, Sicherheit, Audit-Trails und sofortige REST-APIs.

FeatureBeschreibung
TabellenEntsprechen Datenbanktabellen; es gibt Standard- (z.B. Account, Contact) und Custom-Tabellen
SpaltentypenText, Zahl, Datum, Lookup, Choice, Ja/Nein, Datei, Bild, Formel, Rollup uvm.
Beziehungen1:N, N:N und Hierarchie-Beziehungen direkt in der Oberfläche konfigurierbar
Business RulesValidierungsregeln und Feld-Sichtbarkeit ohne Code direkt auf Tabelle definieren
Plugins / Cloud FlowsServerside-Logik bei Datensatz-Ereignissen (Create, Update, Delete)
SicherheitsrollenGranulare Zugriffssteuerung auf Tabellen-, Zeilen- und Spaltenebene

Custom Connectors
Eigene REST-APIs einbinden
Fortgeschritten
▼

Wenn kein vorgefertigter Connector existiert, kann man jede REST-API als Custom Connector einbinden. Basis ist eine OpenAPI (Swagger)-Beschreibung.

  • 1
    OpenAPI-Datei importieren oder manuell definieren
    Endpunkte, Parameter und Auth-Typ (API Key, OAuth 2.0, Basic) festlegen
  • 2
    Verbindung testen
    Im Custom-Connector-Portal direkt API-Calls absetzen
  • 3
    In App / Flow verwenden
    Custom Connector erscheint dann in der Connector-Auswahl wie ein nativer Connector
💡
Mit dem HTTP-Connector (Premium) kann man auch ohne Custom Connector direkt REST-Calls machen – ideal für schnelle Prototypen oder einmalige API-Abfragen.

Wie kombiniert man die Tools? Typische End-to-End-Lösungen und Architekturempfehlungen.

Szenario 1: Digitaler Genehmigungsprozess
Urlaubsantrag ohne Papier – End to End
Praxis
▼
