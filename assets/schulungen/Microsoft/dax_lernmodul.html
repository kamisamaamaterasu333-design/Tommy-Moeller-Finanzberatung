Grundlagen

Was ist DAX, wie unterscheidet es sich von Excel, und welche Konzepte sind zentral?

Was ist DAX?
Data Analysis Expressions – die Formelsprache von Power BI
Einstieg
▼

DAX steht für Data Analysis Expressions. Es ist die Formelsprache in Power BI (und auch Excel Power Pivot / SSAS). DAX wird verwendet, um berechnete Spalten, Measures und berechnete Tabellen zu erstellen.

💡
Measure vs. berechnete Spalte: Ein Measure wird dynamisch berechnet, je nach dem aktuellen Filterkontext (Slicer, Visuals). Eine berechnete Spalte wird beim Laden einmal berechnet und in der Tabelle gespeichert. In 90 % der Fälle sind Measures die bessere Wahl.
Die wichtigsten Konzepte
KonzeptErklärung
FilterkontextWelche Daten aktuell "sichtbar" sind – bestimmt durch Slicer, Zeile/Spalte in Matrix, visuelle Filter
ZeilenkontextExistiert in berechneten Spalten und Iteratoren (X-Funktionen) – bezieht sich auf die aktuelle Zeile
KontexttransitionCALCULATE wandelt Zeilenkontext in Filterkontext um – sehr wichtig bei Iteratoren
Tabellen[Spalte]Standard-Syntax für Spaltenverweise in DAX
[Measure Name]Measures werden immer ohne Tabellennamen referenziert
✅
Schreibe Measures immer in eine eigene Measures-Tabelle (leere Tabelle erstellen → Measures per "Neues Measure" hinzufügen). Das hält das Modell sauber.

Basis-Aggregation

SUM, COUNT, AVERAGE, MIN, MAX, DIVIDE – die Grundbausteine jedes Reports.

SUM
Alle Werte einer Spalte summieren
EinfachAggregation
▼
Syntax
SUM(Tabelle[Spalte])
Beispiel
-- Gesamtumsatz Total Umsatz = SUM(fct_Sales[Betrag])
Eingabe (Tabelle)
Jan: 5.000 €
Feb: 3.200 €
Mär: 7.800 €
Ergebnis
16.000 €
Summe aller sichtbaren Zeilen
💡
SUM ignoriert automatisch leere Zellen (BLANK). Du musst dafür keine extra Prüfung einbauen.

Werte, nicht-leere Einträge oder Zeilen zählen

▼

FunktionZähltWann verwenden

CALCULATE

Die mächtigste Funktion in DAX – sie verändert den Filterkontext einer Berechnung.

CALCULATE – Grundprinzip
Filter ändern, hinzufügen oder entfernen
MittelKernfunktion
▼
Syntax
CALCULATE(Ausdruck, Filter1 [, Filter2, ...])

CALCULATE wertet den Ausdruck (meist ein Measure) innerhalb eines veränderten Filterkontexts aus. Die angegebenen Filter ersetzen oder erweitern den bestehenden Kontext.

Beispiel 1 – Nach Kategorie filtern
-- Umsatz nur für die Kategorie "Elektronik" Umsatz Elektronik = CALCULATE( [Total Umsatz], dim_Produkt[Kategorie] = "Elektronik" )
Kontext: Alle Produkte
Elektronik: 50.000 €
Kleidung: 30.000 €
Möbel: 20.000 €
Ergebnis (immer)
50.000 €
Unabhängig vom Slicer-Filter
Beispiel 2 – Mehrere Filter
-- Umsatz: Elektronik UND Jahr 2024 Umsatz Elek 2024 = CALCULATE( [Total Umsatz], dim_Produkt[Kategorie] = "Elektronik", dim_Datum[Jahr] = 2024 )
💡
Mehrere Filter in CALCULATE wirken wie ein UND. Für ODER-Filter muss FILTER() mit || kombiniert werden.
CALCULATE mit ALL / ALLEXCEPT
Filter aufheben für Prozentanteile und Vergleiche
Mittel
▼

ALL(Tabelle) entfernt alle Filter auf eine Tabelle. ALLEXCEPT(Tabelle, Spalte1, …) entfernt alle Filter außer den angegebenen Spalten. Klassischer Einsatz: %-Anteil vom Gesamten.

-- Anteil am Gesamtumsatz Anteil % = DIVIDE( [Total Umsatz], CALCULATE([Total Umsatz], ALL(fct_Sales)) ) -- Anteil innerhalb der Kategorie (Region-Filter bleibt) Anteil Kategorie % = DIVIDE( [Total Umsatz], CALCULATE([Total Umsatz], ALLEXCEPT(dim_Produkt, dim_Produkt[Kategorie])) )
Report zeigt: Region Nord
Elektronik: 15.000 €
Kleidung: 9.000 €
Gesamt Nord: 24.000 €
Anteil % Elektronik
62,5 %
15.000 / 24.000 (ALL hebt Produkt-Filter auf)

Iteratoren (X-Funktionen)

SUMX, AVERAGEX, COUNTX, MAXX, MINX – Zeile für Zeile berechnen, dann aggregieren.

SUMX – Wie Iteratoren funktionieren
Erst pro Zeile berechnen, dann summieren
MittelIterator
▼
Syntax
SUMX(Tabelle, Ausdruck_pro_Zeile)

SUMX geht jede Zeile der Tabelle durch, berechnet den Ausdruck für diese Zeile und summiert anschließend alle Ergebnisse. Ideal wenn die Berechnung erst auf Zeilenebene erfolgen muss (z.B. Preis × Menge).

Beispiel: Umsatz = Preis × Menge
-- FALSCH: SUM(Preis) * SUM(Menge) → falsches Ergebnis! Umsatz Falsch = SUM(fct_Sales[Preis]) * SUM(fct_Sales[Menge]) -- RICHTIG: Erst pro Zeile multiplizieren, dann summieren Umsatz = SUMX( fct_Sales, fct_Sales[Preis] * fct_Sales[Menge] )
Tabelle
Zeile 1: 10 € × 3 = 30 €
Zeile 2: 25 € × 1 = 25 €
Zeile 3: 5 € × 10 = 50 €
SUMX Ergebnis
105 €
30 + 25 + 50 (korrekt!)
⚠️
Iteratoren mit großen Tabellen (> 1 Mio. Zeilen) können langsam werden. Wenn möglich, berechne die Spalte vorab in Power Query.
AVERAGEX, MAXX, MINX, COUNTX
Weitere Iteratoren für verschiedene Aggregationen
Mittel
▼
-- Durchschnittlicher Bestellwert (Preis × Menge pro Bestellung) Ø Bestellwert = AVERAGEX( fct_Sales, fct_Sales[Preis] * fct_Sales[Menge] ) -- Höchster Bestellwert einer einzelnen Zeile Max Bestellwert = MAXX( fct_Sales, fct_Sales[Preis] * fct_Sales[Menge] )

FILTER & ALL-Familie

Tabellen dynamisch filtern, Filter aufheben, Werte auswählen.

FILTER
Tabelle mit Bedingung einschränken
Mittel
▼
Syntax
FILTER(Tabelle, Bedingung)

FILTER gibt eine Teilmenge einer Tabelle zurück. Es wird meist als Argument in CALCULATE verwendet, wenn der Filter komplex ist (z.B. mit ODER oder Vergleichsoperatoren auf Measures).

-- Umsatz nur für Bestellungen über 500 € Umsatz Grosskunden = CALCULATE( [Total Umsatz], FILTER( fct_Sales, fct_Sales[Betrag] > 500 ) ) -- ODER-Filter: Kategorie A ODER B Umsatz A oder B = CALCULATE( [Total Umsatz], FILTER( dim_Produkt, dim_Produkt[Kategorie] = "A" || dim_Produkt[Kategorie] = "B" ) )
SELECTEDVALUE & VALUES
Aktuellen Slicer-Wert auslesen
Mittel
▼
-- Aktuell ausgewähltes Jahr im Slicer Ausgewähltes Jahr = SELECTEDVALUE(dim_Datum[Jahr], "Alle Jahre") -- Dynamischer Titel Report Titel = "Umsatz " & SELECTEDVALUE(dim_Datum[Jahr], "(alle Jahre)")
💡
SELECTEDVALUE gibt BLANK zurück (oder den Fallback-Wert), wenn mehr als ein Wert ausgewählt ist. Ideal für dynamische Titel und bedingte Anzeige in Visuals.

Time Intelligence

Zeitvergleiche: YTD, Vorjahr, gleitende Durchschnitte. Voraussetzung: eine korrekte Datumstabelle!

⚠️
Pflicht: Datumstabelle! Alle Time-Intelligence-Funktionen benötigen eine vollständige, zusammenhängende Datumstabelle (kein einziges Datum darf fehlen), die als "Datumstabelle" markiert ist (Tabellenwerkzeuge → Als Datumstabelle markieren).
TOTALYTD – Year to Date
Kumulierter Wert vom Jahresanfang bis heute
MittelTime Intelligence
▼
-- Umsatz YTD (Standard: 1. Januar) Umsatz YTD = TOTALYTD([Total Umsatz], dim_Datum[Datum]) -- Für abweichendes Geschäftsjahr (endet 31. März) Umsatz GJ-YTD = TOTALYTD([Total Umsatz], dim_Datum[Datum], "03-31")
Monatswerte 2024
Jan: 10.000 €
Feb: 8.000 €
Mär: 12.000 €
YTD im März
30.000 €
10.000 + 8.000 + 12.000
SAMEPERIODLASTYEAR & DATEADD
Vorjahresvergleich (PY) und flexibler Zeitversatz
MittelTime Intelligence
▼
-- Umsatz Vorjahr (gleicher Zeitraum) Umsatz PY = CALCULATE( [Total Umsatz], SAMEPERIODLASTYEAR(dim_Datum[Datum]) ) -- Veränderung zum Vorjahr absolut Δ vs. Vorjahr = [Total Umsatz] - [Umsatz PY] -- Veränderung zum Vorjahr in % Δ% vs. Vorjahr = DIVIDE([Δ vs. Vorjahr], [Umsatz PY]) -- Flexibel: Vormonat (DATEADD) Umsatz Vormonat = CALCULATE( [Total Umsatz], DATEADD(dim_Datum[Datum], -1, MONTH) )
💡
DATEADD ist flexibler als SAMEPERIODLASTYEAR. Du kannst -1 bis -N Perioden zurückgehen und als Einheit DAY, MONTH, QUARTER oder YEAR wählen.
DATESYTD / DATESMTD / DATESQTD
YTD, MTD, QTD mit CALCULATE kombiniert
Mittel
▼
-- Month to Date (MTD) Umsatz MTD = CALCULATE([Total Umsatz], DATESMTD(dim_Datum[Datum])) -- Quarter to Date (QTD) Umsatz QTD = CALCULATE([Total Umsatz], DATESQTD(dim_Datum[Datum])) -- YTD Vorjahr (kombiniert) Umsatz PY-YTD = CALCULATE( [Total Umsatz], DATESYTD(SAMEPERIODLASTYEAR(dim_Datum[Datum])) )

IF, SWITCH & Logik

Bedingte Berechnungen, Kategorisierungen und Ampel-Logik.

IF
Einfache Wenn-Dann-Sonst-Bedingung
Einfach
▼
IF(Bedingung, Wert_wenn_wahr, Wert_wenn_falsch)
-- Ampel-Logik: Ziel erreicht? Status = IF( [Total Umsatz] >= [Umsatz Ziel], "✅ Ziel erreicht", "❌ Ziel verfehlt" ) -- Verschachtelt: 3 Stufen Bewertung = IF([Marge %] >= 0.3, "Gut", IF([Marge %] >= 0.15, "OK", "Kritisch" ) )
💡
Bei mehr als 2 Stufen: lieber SWITCH verwenden – lesbarer und performanter als tief verschachtelte IFs.
SWITCH & SWITCH(TRUE())
Mehrere Fälle elegant handhaben
Einfach
▼
-- SWITCH für feste Werte Region Name = SWITCH(fct_Sales[RegionID], 1, "Nord", 2, "Süd", 3, "West", "Unbekannt" -- Standardwert ) -- SWITCH(TRUE()) für Bereiche – ersetzt verschachtelte IFs Umsatz Klasse = SWITCH(TRUE(), [Total Umsatz] >= 100000, "A – Großkunde", [Total Umsatz] >= 50000, "B – Mittelkunde", [Total Umsatz] >= 10000, "C – Kleinkunde", "D – Neukunde" )
IFERROR, ISBLANK & BLANK()
Fehler abfangen und leere Werte behandeln
Einfach
▼
-- Fehler abfangen Sicherer Wert = IFERROR([Komplexes Measure], 0) -- Leere Zellen explizit zurückgeben (unterdrückt 0-Linien in Charts) Kein Wert = IF([Total Umsatz] = 0, BLANK(), [Total Umsatz]) -- Prüfen ob Wert leer Hat Umsatz = IF(ISBLANK([Total Umsatz]), "Kein Umsatz", "Hat Umsatz")
💡
BLANK() vs. 0: BLANK lässt einen Datenpunkt im Diagramm leer (keine Linie), 0 zeigt eine Null-Linie. Wähle je nach Darstellungsziel.

Text-Funktionen

Texte zusammenfügen, formatieren, trennen – oft in berechneten Spalten verwendet.

CONCATENATE, & und FORMAT
Texte verbinden und Zahlen formatieren
Einfach
▼
-- & ist kürzer als CONCATENATE Vollständiger Name = dim_Kunden[Vorname] & " " & dim_Kunden[Nachname] -- FORMAT: Zahl als formatierten Text Umsatz Text = FORMAT([Total Umsatz], "#.##0 €") -- Datum formatieren Datum Text = FORMAT(TODAY(), "DD.MM.YYYY") -- Dynamischer Titel mit Formatierung KPI Titel = "Umsatz: " & FORMAT([Total Umsatz], "#.##0 €") & " | Stand: " & FORMAT(TODAY(), "MMMM YYYY")
LEFT, RIGHT, MID, LEN, TRIM
Textteile extrahieren und bereinigen
Einfach
▼
-- Erste 3 Zeichen (z.B. Kürzel) Kürzel = LEFT(dim_Produkt[Name], 3) -- Leerzeichen entfernen Name Bereinigt = TRIM(dim_Kunden[Name]) -- Enthält? (SEARCH gibt Position zurück, IFERROR fängt "nicht gefunden") Enthält GmbH = IF(IFERROR(SEARCH("GmbH", dim_Kunden[Firma]), 0) > 0, TRUE, FALSE)

VAR & RETURN

Measures lesbarer und performanter schreiben durch Zwischenwerte.

VAR & RETURN – Grundprinzip
Zwischenergebnisse speichern statt wiederholen
MittelBest Practice
▼

VAR erlaubt es, Teilberechnungen in benannten Variablen zu speichern. Das macht Measures lesbarer und schneller (da jede VAR nur einmal berechnet wird).

Ohne VAR (unleserlich, langsam)
-- Schlechtes Beispiel: Gleicher Ausdruck mehrfach Δ% = DIVIDE( [Total Umsatz] - CALCULATE([Total Umsatz], SAMEPERIODLASTYEAR(dim_Datum[Datum])), CALCULATE([Total Umsatz], SAMEPERIODLASTYEAR(dim_Datum[Datum])) )
Mit VAR (lesbar, einmal berechnet)
VAR AktuellerUmsatz = [Total Umsatz] VAR VorjahrUmsatz = CALCULATE([Total Umsatz], SAMEPERIODLASTYEAR(dim_Datum[Datum])) VAR Differenz = AktuellerUmsatz - VorjahrUmsatz RETURN DIVIDE(Differenz, VorjahrUmsatz)
✅
Verwende VAR & RETURN immer wenn du denselben Teilausdruck mehr als einmal brauchst – das ist eine der wichtigsten Best Practices in DAX.
Weiteres Beispiel: Bedingte Logik mit Zwischenwerten
VAR Marge = DIVIDE([Gewinn], [Total Umsatz]) VAR Ziel = 0.25 VAR Abweichung = Marge - Ziel RETURN SWITCH(TRUE(), Abweichung >= 0.05, "🟢 Sehr gut", Abweichung >= 0, "🟡 Im Ziel", Abweichung >= -0.05, "🟠 Leicht darunter", "🔴 Kritisch" )

Ranking & Top N

RANKX, TOPN und dynamische Top-N-Filter.

RANKX
Rang eines Wertes innerhalb einer Gruppe ermitteln
Fortgeschritten
▼
Syntax
RANKX(Tabelle, Ausdruck [, Wert, Reihenfolge, Gleichstand])
-- Rang der aktuellen Kategorie nach Umsatz Rang Umsatz = RANKX( ALL(dim_Produkt[Kategorie]), -- Alle Kategorien vergleichen [Total Umsatz], -- Nach diesem Wert ranken , -- Leer = aktueller Wert DESC, -- Höchster = Rang 1 DENSE -- Keine Lücken bei Gleichstand ) -- Nur Top 5 anzeigen (für bedingte Formatierung) Ist Top 5 = IF([Rang Umsatz] <= 5, TRUE, FALSE)
Kategorien (nach Umsatz)
Elektronik: 50.000 € → Rang 1
Möbel: 30.000 € → Rang 2
Kleidung: 20.000 € → Rang 3
Rang Kleidung
3
DESC + DENSE
TOPN & dynamisches Top N
Die besten N Einträge filtern
Fortgeschritten
▼
-- Umsatz der Top-3-Produkte Top 3 Umsatz = CALCULATE( [Total Umsatz], TOPN(3, ALL(dim_Produkt[Produkt]), [Total Umsatz]) ) -- Dynamisch: Top N via What-If-Parameter (N aus Slicer) Top N Umsatz = VAR N = SELECTEDVALUE(TopN_Parameter[Wert], 5) RETURN CALCULATE( [Total Umsatz], TOPN(N, ALL(dim_Produkt[Produkt]), [Total Umsatz]) )
💡
Für ein dynamisches Top-N erstelle in Power BI einen What-If-Parameter (Modellierung → Neuer Parameter). Dieser generiert automatisch eine Tabelle und einen Slicer.
